# Политика версий API

## Общие принципы
- API версионируется по URL: `/api/v1/*`.
- Изменения без обратной совместимости сопровождаются выпуском новой версии (`v2`).

## Допустимые изменения в рамках v1
- Добавление новых эндпоинтов.
- Добавление новых полей в ответах.
- Добавление новых необязательных полей в запросах.

## Недопустимые изменения в рамках v1
- Удаление полей из ответов.
- Изменение типов полей.
- Изменение семантики без миграционного пути.

## Депрекейты
- Депрекейт объявляется в релиз‑заметках.
- Депрекейт отмечается в `docs/openapi.yaml`.
- Переходный период — не менее 1 релиза.
