# Обновление версий

## Перед обновлением
- Прочитать релиз‑ноты и проверить совместимость API.
- Проверить состояние БД и наличие свежего бэкапа.
- Убедиться, что есть доступ к registry образов и секретам.

## Порядок обновления (Kubernetes)
1. Обновить теги образов в манифестах/Helm.
2. Запустить миграции (`cmd/migrate`) до выката API.
3. Применить манифесты/Helm и дождаться rollout.
4. Проверить `/healthz`, ключевые сценарии, метрики.

## Rollback
- Если проблемы — использовать `docs/rollback.md`.
- При необходимости — восстановление из бэкапа (см. `docs/disaster-recovery.md`).
